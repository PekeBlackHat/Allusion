# Security Policy

## Supported Versions

Allusion is an actively developed research tool. Security fixes will generally be applied to the latest version of the project.

| Version        | Supported |
| -------------- | --------- |
| Latest Release | ✅         |
| Older Releases | ❌         |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

Please include:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Suggested remediation (if known)

Avoid publicly disclosing vulnerabilities before they have been reviewed.

---

## Scope

Allusion is designed as a privacy-friendly public web research tool.

The project is intended to:

* Search publicly available web content
* Retrieve publicly accessible webpages
* Extract readable text from public sources
* Generate exploratory research reports

The project is **not designed** to:

* Access private accounts
* Access email systems
* Access calendars
* Access contact lists
* Access private files
* Circumvent authentication
* Bypass paywalls
* Access content behind logins
* Perform credential harvesting
* Conduct unauthorized surveillance

---

## Privacy Considerations

Allusion operates exclusively on:

* User-provided topics
* Public search results
* Publicly accessible webpages

No personal user data is collected, stored, sold, or transmitted by the project itself.

Generated reports may contain information obtained from publicly available sources. Users are responsible for reviewing source material and complying with applicable laws and terms of service.

---

## Security Design Principles

Allusion follows several core security principles:

### Public-Only Research

Allusion is intended for public information discovery and research exploration.

### Read-Only Behavior

The tool does not modify remote systems, accounts, databases, or third-party services.

### No Credential Collection

The project does not request, store, process, or transmit passwords, authentication tokens, API keys, or private credentials.

### Transparent Source Attribution

Reports include source references so users can independently verify findings.

### User Verification Required

Generated observations and analytical findings may be incomplete, incorrect, or speculative. Users should verify conclusions against the underlying sources before relying on them.

---

## Known Limitations

Allusion is a research-assistance tool, not an authoritative source of truth.

Potential limitations include:

* Incomplete source coverage
* Search engine bias
* Source quality variability
* Incorrect pattern detection
* False correlations
* Hallucinated interpretations from analytical synthesis components

Users should treat reports as research leads rather than established facts.

## Responsible Use

Users are expected to comply with:

* Applicable laws and regulations
* Website terms of service

* Research ethics standards

The maintainers do not endorse misuse of the software for unauthorized access, surveillance, or illegal activities.
