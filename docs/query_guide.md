# Query Guide

# Writing Effective Allusion Queries

The quality of an Allusion report depends heavily on the query.

Allusion is designed to explore relationships, tensions, emerging ideas, and underexplored areas. Broad queries often produce broad reports. Specific and intentional queries produce better discoveries.

---

# General Rule

A good query combines:

```text
Domain
+
Method
+
Goal
```

Example:

```text
phase amplitude coupling
+
machine learning
+
EEG classification
```

Produces:

```text
phase amplitude coupling machine learning EEG classification
```

---

Example:



Goal:

* Build a research map
* Understand major themes
* Identify recurring concepts

# Poor Queries

Avoid:

```text
AI
```

```text
machine learning
```

```text
agents
```

```text
EEG
```

These are too broad.

They produce weak reports because they lack context and direction.

---

# Strong Queries

Prefer:

```text
phase amplitude coupling statistical significance EEG classification
```

```text
underexplored AI agent architectures
```

```text
open ended evolution reinforcement learning autonomous agents
```

```text
evolving neural agents in open ended worlds
```

These queries provide enough structure for Allusion to identify meaningful patterns.

---

# Philosophy

Allusion works best when exploring:

* Emerging ideas
* Underexplored directions
* Hidden connections
* Technical niches
* Research frontiers

A useful query should make you think:

> I don't know what I'll find, but if there is a signal here, I want to see it.

# Query Design Principle

The strongest Allusion queries usually contain one or more of:

```text
underexplored
emerging
experimental
open ended
autonomous
self improving
alternative
forgotten
prototype
novel
```

These words push the search toward discovery rather than popularity.

A good Allusion query should feel less like:

"What is this?"

and more like:

"What are people missing?"
